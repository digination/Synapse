Synapse Container (SynContainer)

# of inputs: 1
# of outputs: 1
Resizable: Yes

Behaviour:

As its name says, a Synapse container is a block that can contain other building blocks ( included other containers ). Since each container can be played independantly from the workfow they're in, we can consider containers as "subworkflows" It is useful to split workfows logically into subparts.

Building Block Options:

Name

type: String
Default Value: Container#

Simply defines the blocks name. Also please note that a block name is not an UID ( meaning that you can have multiple blocks having the same name )

Color

type: HexString
Default Value: None

defines the building blocks color in the scheme
