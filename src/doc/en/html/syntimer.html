Synapse Timer (SynTimer)

# of inputs: 1
# of outputs: 1
Resizable: No

Behaviour:

When the syntimer building block receives data on its input buffer, it will wait a specific amount of time before releasing it on its output.

Notes on syntimer: Like any other time-based component in synapse, it has a 50ms resolution ( can't do much better because of python limitations). Also, take into consideration that the syntimer input buffer is a queue, meaning that if it recieves some data while there is already some data waiting to be released, it will not matter and the additional data will be added to the queue. When the counter reaches "Period" milliseconds, it will then restart to count instantly to release the second element in the queue.

Building Block Options:

Name

type: String
Default Value: Timer#

Simply defines the blocks name. Also please note that a block name is not an UID ( meaning that you can have multiple blocks having the same name )

Color

type: HexString
Default Value: None

defines the building blocks color in the scheme

Period

type: Integer
Default Value: 0

Specifies the amount of time (in milliseconds) the syntimer must wait until being able to release its output buffer.
